# Info-Helper — утренние поводы для касания клиентов

> Кейс/демо из сессии AI Camp Almaty 2026 · Глава «Менеджер» · 0:15–0:22

## Проблема

Что бы делал хороший менеджер каждое утро, если бы у него было время? Открыл бы новости по своим клиентам. Кто сменил позицию. Какие у них события. Какие пресс-релизы.

**Никто этого не делает.** Это час времени в день, а у менеджера этого часа нет.

## Результат на реальном кейсе

IT-провайдер в Европе, B2B, ~200 активных клиентов в воронке. Простой сценарий: CRM + источники свежих новостей → Claude → Telegram. Менеджер каждое утро в 9:00 получает **3-5 актуальных поводов** для касания.

**Talk time у менеджера +22%.** Не быстрее звонит — каждое утро готовый повод позвонить конкретному клиенту, без research-а.

## Две реализации

### 🟢 [skill/](skill/) — Production-инструмент (Claude/Cursor Skill + cron)

**Это то, что ставишь себе в реальную работу.**

- Запускается каждое утро в 9:00 без участия человека.
- Конфиг списка клиентов в YAML, дедупликация между запусками, многоканальная доставка (Telegram / Slack / Email / Markdown).
- Стек: только Anthropic API (~$10/мес на 50 клиентов). Опционально — Apify (LinkedIn), xAI Grok (X/Twitter).
- Установка: 10 минут (clone → `.env` → `cron`).
- Полная инструкция → [skill/README.md](skill/README.md).

### 🟣 [workshop/](workshop/) — Демо на сцене (Make.com)

**Это не для продакшена. Это объяснение архитектуры через визуальный сценарий.**

- 5 нод в Make, drag-and-drop.
- На сцене показывает: «вот как это просто соединить — Cron → CRM → Perplexity → Claude → Telegram».
- Если хочешь повторить демо у себя — импортируешь blueprint за 4 минуты.
- Для производственной работы → переходи на skill-ветку.
